Det er præcis det rigtige sted at bevise en ide. Men det er det forkerte sted at drive en forretning.

✕Ingen brugerkonti — ingen der ejer noget
✕Ingen sessions — alt forsvinder efter brug
✕Ingen betaling — nul revenue-mulighed
✕Ingen branding — det er Googles interface
✕Ingen skalering — manuelt for hvert shoot
✕Ingen data — vi ved ikke hvem der bruger det
✕Kan ikke deles med kunder som produkt

Slide 04 — The Bridge

Et lag platform — og vi er et rigtigt produkt.

Vi bringer Google AI API'en ud og wrapper en fuld SaaS-forretning rundt om den. Det er præcis hvad Runway, Midjourney og Adobe Firefly har gjort.

Nu — Google AI Studio
Manual session hvert enkelt gang
Kun intern adgang — ingen rigtige kunder
Googles branding og UI — ikke vores
Nul revenue — ingen betalingsflow
Ingen historik — shoots gemmes ikke
Target — Hummel Creative Studio Platform
Brugerlogin — alle kunder har egne konti
Stripe betalinger — credits og subscriptions
Projekt-storage — alle shoots gemt permanent
Hummel UI — vores brand, vores flow
Analytics — vi ved hvad kunderne laver
💡 Den store indsigt: Vi opfinder ikke AI'en igen. Vi bygger den forretning rundt om den, som de store spillere har gjort. Runway og Midjourney er bygget på eksisterende AI-APIs — ikke proprietary models.

Slide 12 — Migrering

Fra Google AI Studio → Claude. Sådan gør vi det.

Vi eksporterer det vi har bygget, konverterer det til Claude-format og kører det i vores egen kodebase. 3 trin.

Trin 1 — Eksportér
  • Google AI Studio → Mine prompts
  • Download system-prompts som .txt
  • Notér model-indstillinger
    (temperature, top-p, max tokens)
  • Gem alle eksempel input/output par
  • Alt du har bygget er reddet
Trin 2 — Konvertér
Google AI format:
System: “You are a photo studio...”
User: “Generate shoot for [model]”
Claude format (prompts.ts):
const systemPrompt = `
  You are an expert AI
  fashion photographer...
`
Trin 3 — Validér
  • Kør de samme test-cases i Claude som Google AI
  • Sammenlign output kvalitet side om side
  • Claude 200k kontekst → bedre end Google AI
  • Justér system-prompt til output er bedre
  • Hele processen: 2–3 dage

Slide 13 — Claude Arkitektur

Koden der binder alt sammen.

Anthropic TypeScript SDK — 10 linjer kode erstatter hele Google AI Studio-sessionen.

lib/claude.ts — API-klienten
import Anthropic from '@anthropic-ai/sdk'

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY
})

export async function generateShoot(prompt) {
  return await client.messages.create({
    model: 'claude-3-5-sonnet-20241022',
    system: systemPrompt,
    messages: [{ role: 'user', content: prompt }]
  })
}
⚡️ Streaming — ingen spinner
  • Svar vises token for token i realtid
  • Reducerer oplevet ventetid med ~70%
  • Samme følelse som ChatGPT
🔒 Sikkerhed fra dag 1
  • API-nøgle kun i .env — aldrig i koden
  • Sæt spending limit på Anthropic dashboard
  • Log kald med userId — aldrig med nøglen
📦 Installation
npm install @anthropic-ai/sdk
Én pakke. Ingen opsætning. Virker.
💡 Hele Google AI Studio-sessionen erstattes af én lib/claude.ts fil og en .env fil. Det er det. Vi ejer nu hele kaldet.
